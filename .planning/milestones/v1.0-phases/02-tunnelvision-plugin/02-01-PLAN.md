## Plugin Spec Conventions (from leap.lua and snacks.lua)

Each plugin under `nvim/.config/nvim/lua/illico/plugins/` returns a table of lazy.nvim spec dicts. Pattern:

```lua
return {
  {
    "author/plugin.nvim",
    -- optional: config = function() ... end
    -- optional: keys = { {...}, {...} }
    -- optional: opts = { ... }
  },
}
```

Keys lazy-load syntax (from snacks.lua):
```lua
keys = {
  {
    "<leader>KEY",
    function() ... end,
    desc = "Description for which-key",
  },
}
```

## Tunnelvision API (from CONTEXT.md)

Verified from official README:
- `setup(opts)` accepts `{ mode, scope, source }` or `{}`
- Commands: `:TunnelVision on|off|toggle|next|prev|retarget|refresh|status`
- Lua API: `require("tunnelvision").on(opts)`, `.off()`, `.is_active()`
- **Toggle action:** PREFER `:TunnelVision toggle` (robust, documented); fallback to Lua wrapper with `is_active()` if command unavailable

## Executor's Responsibility

Before finalizing the toggle action in step 2:
1. Check `~/.local/share/nvim/lazy/tunnelvision.nvim/` for installed plugin source
2. Verify whether `:TunnelVision toggle` command actually exists (grep for `command!.*TunnelVision` in plugin source)
3. If found: use `<cmd>TunnelVision toggle<CR>`
4. If NOT found: fall back to Lua wrapper:
   ```lua
   function()
     local tv = require("tunnelvision")
     if tv.is_active() then tv.off() else tv.on() end
   end
   ```
5. Document which approach was used in the SUMMARY
